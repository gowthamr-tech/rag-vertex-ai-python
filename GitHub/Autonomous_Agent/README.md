# Autonomous Build/Review/PR Agent

A lightweight autonomous coding agent you can run in any repo to:
- detect your stack,
- run build/test/lint checks,
- review diffs for common risk patterns,
- execute objective-driven loops,
- create a PR automatically.

## Quick start

```bash
python3 autonomous_agent.py run
```

This generates a review report at:

```text
.agent/review.md
```

## Commands

### 1) Build + review

```bash
python3 autonomous_agent.py run
```

### 2) Review only (no build/test/lint)

```bash
python3 autonomous_agent.py review
```

### 3) Build + review + create PR

```bash
python3 autonomous_agent.py run-and-pr \
  --base main \
  --title "feat: autonomous improvements" \
  --body "Automated PR generated by autonomous agent" \
  --commit-message "feat: autonomous improvements"
```

### 4) Autonomous objective mode (planner + retries + report)

```bash
python3 autonomous_agent.py autonomous \
  --objective "stabilize tests and lint" \
  --max-iterations 2
```

Optional LLM planning (requires `OPENAI_API_KEY`):

```bash
python3 autonomous_agent.py autonomous \
  --objective "validate build and security checks" \
  --use-llm \
  --model gpt-4.1-mini
```

Autonomous mode writes:
- `.agent/review.md`
- `.agent/state.json`

Open a PR in autonomous mode:

```bash
python3 autonomous_agent.py autonomous \
  --objective "run full quality gate" \
  --open-pr \
  --base main
```

## Stack detection

- Node.js: `package.json`
- Python: `pyproject.toml`, `requirements.txt`, or `setup.py`
- Go: `go.mod`
- Rust: `Cargo.toml`
- Fallback: no inferred workflow (agent reports that build/test/lint is not auto-detected)

## Safety model

- Agent commands are restricted to a command allowlist (build/test/lint/git tooling).
- Unsafe generated commands are blocked.
- Basic automatic recovery attempts run on common failures (for example missing `pytest`).

## Notes

- PR creation requires GitHub CLI (`gh`) authenticated.
- Branches are created with `codex/agent-<timestamp>` prefix.
- Review checks include basic heuristics (debug logs, TODO/FIXME, bare except, eval usage, possible credential strings).
